🔁 Common Actions — available across multiple flows

These actions are not tied to one state — they appear alongside primary CTAs whenever their conditions are met.

👤 Assign Technician / Assign Self

INSTALL + SERVICE

CSP can delegate work to a team member or take it themselves. Opens a bottom sheet with a radio list of available technicians + "Self" option.

  • Assign Self → CSP will do the work personally. Card transitions to a state where CSP can start working.
  • Assign Technician → Delegated to team member. Card moves to "Team assigned" (In Motion).
FlowAvailable in states
InstallSlot Confirmed, Delegate Overdue
ServicePending Acceptance, Accepted, Respond Now
Pickup— not available
Recharge— not available

📞 Call Customer (Masked)

INSTALL + SERVICE + PICKUP + NBREC

CSP can call the customer via a masked number (CSP's real number is hidden). Fire-and-forget — no state change. Tertiary button with phone icon.

  • Condition: maskedCallAvailable == true
  • Install extra gate: Only on slot day (isSlotDay == true)
  • Also available as inline icon next to technician name in delegated states
FlowAvailable in states
InstallScheduled, In Progress, Team Assigned, Team Working, Delegate Overdue (slot day only)
ServiceWorking, Escalated, Customer Denied, Team Assigned, Team Working
PickupAll states with drilldown open
Recharge— not available

🚪 Exit Task

INSTALL only

CSP can drop an installation task entirely. Opens a reason sheet with 4 structured reasons. Destructive — requires confirmation.

  • Reasons: Location unreachable, Capacity full, Customer cancelled, Technician unavailable
  • Early state (Propose Slot): Exit button is inline, easy to reach — low investment
  • Later states: Exit hidden in overflow menu (⋯) — protects against accidental exit
  • Card removed from feed after confirmation
FlowAvailable in states
InstallAll states where exitAvailable == true
Service— not available (no exit mechanism)
Pickup— uses "Report Unable" instead
Recharge— not available

🚫 Report Unable to Complete

PICKUP only

CSP can report they cannot complete a pickup. Opens a reason sheet with 5 structured reasons + optional free text for "Other".

  • Reasons: Device not found, Customer not available, Location inaccessible, Device damaged, Other
  • "Other" unlocks a text input (3–200 chars)
  • Card removed from feed after confirmation
  • Appears as a tertiary text link, not primary CTA
FlowAvailable in states
PickupSubmit Photos (S03), Re-submit Photos (S04)
Install— uses "Exit Task" instead
Service— not available

Proof Steps (System / Customer)
प्रूफ सत्यापन चरण
NORMAL
After tapping Resolve, a guided proof flow appears. One step at a time — CSP triggers each verification step sequentially.
Proof steps in order

Step 1: Device Event (automatic — runs in background, no CSP action).
Step 2: System Check — CSP taps to trigger system verification.
Step 3: Customer Confirm — only if System Check fails. CSP taps to request customer verification.

Trigger System Check Request Customer Confirm
