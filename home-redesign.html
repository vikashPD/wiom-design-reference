Step 3 · The solution

Single feed — latest update on top, red dot for action

One unified feed sorted by most recent update. Cards needing CSP action show a red dot — the dot stays until the CSP acts. All cards (pending, in-process, completed) live in the same feed and naturally drift down when they have no new updates.

Home Feed — Sequence Logic

How cards are ordered, when they move, and how long they stay

Core rule: The home feed is a single sorted list. Every card — pending, in-process, or completed — lives here. The card with the most recent update moves to the top. Cards with no new updates naturally drift down as other cards get updated above them.

1. Latest update = top position

Whenever a card receives a new update (state change, customer action, system event), it jumps to the top of the feed. This applies to all card types equally — Install, Service, Pickup, Shifting, and Recharge.

2. Red dot = action needed

If the update requires CSP action (assign tech, propose slot, verify, recharge), the card shows a red dot. The red dot stays until the CSP completes the action. Cards without a red dot are FYI — CSP can see them but doesn't need to act.

3. 7-day retention

All cards stay on the home feed for 7 days from creation. After 7 days, the card is removed automatically. This duration is controlled by the backend and can be changed without an app update.

4. Natural decay

Completed and in-process cards with no new updates automatically drift down the feed over time — because other cards with fresh updates keep pushing above them. No manual sorting needed.

Batch cards (NetBox Pickup & ISP Recharge) behave like all other cards. They are no longer pinned or treated specially. They receive updates, move to the top when updated, show a red dot when action is needed, and drift down when idle — exactly like Install, Service, and Shifting cards.

Red dot — action vs FYI updates

How updates on a card affect the red dot indicator

Core rule: Every update that arrives on a card is classified as either Action (CSP must do something) or FYI (just information). Action updates add a red dot to the card. FYI updates move the card to the top (latest update) but show no red dot. The red dot stays until the CSP completes the action.

Implementation feasibility

What changes in the current architecture

Server-driven — no app release needed

  • Feed sorting by updatedAt — cards already have this field from backend
  • Red dot driven by existing isCspActionable flag from backend
  • 7-day retention is backend-controlled — remove cards from API response after 7 days
  • All card types (Install, Service, Pickup, Recharge, Shifting) use the same sort logic

Needs app change

  • Red dot indicator — small red dot on card when isCspActionable = true
  • Sort by updatedAt — single flat LazyColumn sorted by most recent update timestamp
  • Card reorder animation — optional, animate card moving to new position when updated
  • Batch cards simplified — Pickup and Recharge behave like individual cards, no special batch rendering
